【核心類別結構】
────────────────────────────────────────────────────────────

Form1 (主窗體類別)
│
├─ 成員變數
│  ├─ AxMMMarkx64[] m_MMMark[4]      // 四個打標控件
│  ├─ AxMMEditx64[] m_MMEdit[4]      // 四個編輯控件
│  ├─ AxMMStatusx64[] m_MMStatus[4]  // 四個狀態控件
│  ├─ Panel[] m_Panels[4]            // 四個顯示面板
│  ├─ bool m_bInit                   // 初始化標記
│  └─ int m_iCurrentBoard            // 當前選擇的板
│
├─ 事件處理方法
│  ├─ Form1_Load()              // 窗體載入
│  ├─ btnInit_Click()           // 初始化按鈕
│  ├─ btnDrawLine_Click()       // 繪製線段按鈕
│  ├─ btnMark_Click()           // 開始雷射按鈕
│  ├─ btnStop_Click()           // 停止按鈕
│  ├─ btnExit_Click()           // 退出按鈕
│  └─ Form1_FormClosed()        // 窗體關閉
│
└─ 內部邏輯流程
   └─ 初始化流程 → 繪圖流程 → 打標流程 → 清理流程


【資料流程】
────────────────────────────────────────────────────────────

1. 初始化流程：
   ┌─────────┐    ┌──────────┐    ┌──────────┐
   │點擊初始化│ → │創建控件實例│ → │設定工作區域│
   └─────────┘    └──────────┘    └──────────┘
        ↓
   ┌──────────┐    ┌──────────┐
   │註冊事件處理│ → │完成初始化 │
   └──────────┘    └──────────┘

2. 繪製線段流程：
   ┌─────────┐    ┌──────────┐    ┌──────────┐
   │輸入座標參數│ → │選擇目標板 │ → │點擊繪製按鈕│
   └─────────┘    └──────────┘    └──────────┘
        ↓
   ┌──────────┐    ┌──────────┐    ┌──────────┐
   │清除舊內容 │ → │添加線段對象│ → │設定線段參數│
   └──────────┘    └──────────┘    └──────────┘
        ↓
   ┌──────────┐
   │ 重繪顯示  │
   └──────────┘

3. 雷射打標流程：
   ┌─────────┐    ┌──────────┐    ┌──────────┐
   │確認圖形正確│ → │點擊開始雷射│ → │啟動打標引擎│
   └─────────┘    └──────────┘    └──────────┘
        ↓
   ┌──────────┐    ┌──────────┐    ┌──────────┐
   │執行雷射動作│ → │監控打標狀態│ → │完成打標  │
   └──────────┘    └──────────┘    └──────────┘

4. 停止流程：
   ┌─────────┐    ┌──────────┐    ┌──────────┐
   │點擊停止按鈕│ → │發送停止命令│ → │雷射停止運作│
   └─────────┘    └──────────┘    └──────────┘


【控件陣列說明】
────────────────────────────────────────────────────────────

每個晶片板都有三個關聯的控件：

控件陣列索引：0 = 晶片板1, 1 = 晶片板2, 2 = 晶片板3, 3 = 晶片板4

┌──────────┬─────────────┬─────────────────────────┐
│ 控件類型  │   陣列名稱   │         功能說明          │
├──────────┼─────────────┼─────────────────────────┤
│ MMMark   │ m_MMMark[]  │ 主控件：顯示、打標、控制  │
│ MMEdit   │ m_MMEdit[]  │ 編輯控件：添加圖形、設定  │
│ MMStatus │ m_MMStatus[]│ 狀態控件：監控打標狀態    │
└──────────┴─────────────┴─────────────────────────┘


【關鍵 API 對應】
────────────────────────────────────────────────────────────

功能                  API 方法                    說明
─────────────────────────────────────────────────────────
初始化                Initial()                   初始化控件
設定工作區域中心      SetDesktopCenter(x, y)      設定中心點座標
設定工作區域大小      SetDesktopSize(w, h)        設定寬高
設定資料庫            SetActiveDB(id)             設定當前資料庫ID
設定待機模式          MarkStandBy()               進入待機狀態
設定編輯功能          SetCurEditFun(mode)         設定編輯模式
清除所有內容          ClearAll()                  清空圖形
添加圖形對象          AddObj(type)                添加對象(1=線段)
設定對象參數          SetObjPara(idx,para,val)    設定參數
重繪顯示              Redraw()                    刷新顯示
開始打標              StartMarking(mode)          開始雷射(4=非阻塞)
停止打標              StopMarking()               停止雷射
關閉打標              MarkShutdown()              關閉打標引擎
完成並清理            Finish()                    清理資源


【座標系統說明】
────────────────────────────────────────────────────────────

預設座標系統：
  - 原點 (0, 0) 位於工作區域中心
  - X 軸：向右為正
  - Y 軸：向下為正
  - 單位：毫米 (mm)

工作區域預設設定：
  - 中心點：(0, 0)
  - 寬度：100 mm
  - 高度：100 mm
  - 有效範圍：X: -50 ~ +50, Y: -50 ~ +50

座標示意圖：
        Y-
         ↑
         │
   -X ←──┼──→ +X
         │  (0,0)
         ↓
        Y+


【擴展性設計】
────────────────────────────────────────────────────────────

系統採用陣列結構設計，便於：

1. 支援多晶片板：
   - 當前支援 4 個晶片板
   - 可輕易擴展至更多晶片板（修改陣列大小）

2. 統一管理介面：
   - 所有晶片板使用相同的 API
   - 透過索引選擇不同的晶片板

3. 獨立控制：
   - 每個晶片板獨立初始化
   - 每個晶片板獨立操作
   - 互不干擾

4. 資源管理：
   - 統一的初始化流程
   - 統一的清理流程
   - 確保資源正確釋放


【安全機制】
────────────────────────────────────────────────────────────

1. 初始化檢查：
   - 所有操作前檢查 m_bInit 標記
   - 防止未初始化時執行操作

2. 異常處理：
   - Try-Catch 包覆關鍵操作
   - 友善的錯誤提示訊息

3. 資源清理：
   - FormClosed 事件中統一清理
   - 依序關閉：MarkShutdown → Finish

4. 參數驗證：
   - 座標參數解析驗證
   - 防止無效輸入


【效能優化】
────────────────────────────────────────────────────────────

1. 非阻塞模式：
   - StartMarking(4) 使用非阻塞模式
   - 不會凍結使用者介面

2. 按需初始化：
   - 只在需要時初始化控件
   - 節省系統資源

3. 即時顯示：
   - Redraw() 立即更新顯示
   - 提供即時回饋
