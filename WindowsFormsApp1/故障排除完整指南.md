# COM 錯誤 0x80040154 完整故障排除指南

## 🎯 快速解決方案

### 第一步：測試範例程式

**執行：**
```
C:\Program Files (x86)\MarkingMate\sdk\ocx\CSharp_example\VS2017\LoadFile\LoadFile\bin\x64\Release\LoadFile.exe
```

**結果判斷：**
- ✅ **範例程式能運行** → 問題在您的專案 → 跳到「解決方案 B」
- ❌ **範例程式也失敗** → OCX 未註冊 → 繼續「解決方案 A」

---

## 解決方案 A：註冊 OCX（範例程式也失敗時）

### 選項 1：使用自動腳本

1. **以系統管理員身分**執行 `RegisterAndDiagnose.ps1`
   ```powershell
   # 右鍵 → 以系統管理員身分執行 PowerShell
   Set-ExecutionPolicy -ExecutionPolicy Bypass -Scope Process
   .\RegisterAndDiagnose.ps1
   ```

### 選項 2：手動註冊（最可靠）

1. **以系統管理員身分**開啟命令提示字元
   - 按 `Win + X`
   - 選擇「命令提示字元（系統管理員）」

2. **切換到 DLL 目錄**
   ```cmd
   cd "C:\Program Files (x86)\MarkingMate\sdk\ocx\CSharp_example\VS2017\LoadFile\LoadFile\bin\x64\Release"
   ```

3. **註冊每個 DLL**（逐一執行）
   ```cmd
   regsvr32 MMEditx64Lib.dll
   ```
   - 應該會看到：「DllRegisterServer 在 MMEditx64Lib.dll 成功」
   - 點擊「確定」

   ```cmd
   regsvr32 MMMarkx64Lib.dll
   ```
   - 應該會看到：「DllRegisterServer 在 MMMarkx64Lib.dll 成功」
   - 點擊「確定」

   ```cmd
   regsvr32 MMStatusx64Lib.dll
   ```
   - 應該會看到：「DllRegisterServer 在 MMStatusx64Lib.dll 成功」
   - 點擊「確定」

4. **驗證註冊**
   - 重新執行範例程式
   - 應該能正常啟動

## 解決方案 B：修正專案配置（範例程式能運行時）

這表示 OCX 已註冊，問題在於您的專案配置。

### 檢查清單：

#### 1. 確認平台設定
- Visual Studio → 工具列 → 平台 → **必須是 x64**
- **不能是 AnyCPU 或 x86**

#### 2. 確認 DLL 已複製
```powershell
# 執行 DLL 複製腳本
.\CopyDlls_Simple.ps1
```

驗證這些檔案存在於輸出目錄：
```
bin\x64\Debug\
  - AxMMEdit_x64.dll
  - AxMMMark_x64.dll
  - AxMMStatus_x64.dll
  - MMEditx64Lib.dll
  - MMMarkx64Lib.dll
  - MMStatusx64Lib.dll
```

#### 3. 清理並重建專案
```
建置 → 清理方案
建置 → 重建方案
```

#### 4. 檢查引用
在 Visual Studio 中：
1. 右鍵專案 → 屬性
2. 參考 → 確認這 6 個 DLL 都存在且路徑正確

## 常見錯誤和解決方法

### 錯誤 1：「找不到指定的模組」
**原因：** DLL 依賴項缺失
**解決：**
- 安裝 Visual C++ Redistributable 2017 x64
- 檢查 Windows\System32 中是否有所需的 DLL

### 錯誤 2：「存取被拒」
**原因：** 權限不足
**解決：**
- 以系統管理員身分執行所有註冊命令
- 關閉防毒軟體再試

### 錯誤 3：「模組可能與此 Windows 版本不相容」
**原因：** 平台不匹配
**解決：**
- 確認使用 x64 版本的 DLL
- 確認 Windows 是 64 位元系統
- 確認專案編譯為 x64
